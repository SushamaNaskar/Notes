# ESLint
- ESLint is a linting tool that analyzes code to find potential errors, bugs, and bad practices. 
- It helps enforce coding standards and catches issues like unused variables, incorrect syntax, or risky patterns before runtime.

# Linting
- Linting is the automated process of analyzing source code to flag programmatic errors, bugs, and stylistic issues without running the program. 
- The tools that perform this static analysis are called linters, and they help improve code quality, maintain consistency, and catch potential problems early in the development cycle.
