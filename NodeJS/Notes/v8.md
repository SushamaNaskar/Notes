# v8
V8 is Google’s open source high-performance JavaScript and WebAssembly engine, written in C++. It is used in Chrome and in Node.js, among others. It implements ECMAScript and WebAssembly, and runs on Windows, macOS, and Linux systems that use x64, IA-32, or ARM processors. V8 can be embedded into any C++ application.

#  v8 Javascript runtime envirment
- js engine is not something physical, it's just a program written in c++ => https://github.com/v8/v8 , when we write a js code, js engine converts it to binary that the machine understands
- node.js is a c++ application, with v8 embedded into it.

# If v8 executes any js code, why we needed node.js?
- V8 can execute JavaScript (ECMAScript only),but it cannot access files, network, OS, or databases.
- Node.js = V8 + system APIs (fs, http, net, crypto, etc.)
- Node gives JavaScript power to run outside the browser and build backend applications.

## What V8 Can Do
- Execute JavaScript
- Handle variables, loops, functions
- Run ECMAScript features
- Optimize JS code (JIT compilation)

## But V8 alone cannot:
- read/write files
- connect to databases
- create servers
- access OS
- handle networking
- schedule background tasks
- communicate with hardware
- receive HTTP requests

These features are not part of ECMAScript.
